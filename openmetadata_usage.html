💡 Why Column-Level Tags?
Each attention head maps to a column. This enables fine-grained governance, existing data team workflows, and familiar query patterns. Falls back to table-level when column ops fail.
🔄
Governance Flow
1
Bootstrap Topology
Backend creates synthetic OpenMetadata entities: model → schemas → layers → heads
2
Trace Ingestion
AttentionTrace objects ingested as lineage edges: prompt → layer → head → token
3
Operator Tags Head
User marks a head as DEFECTIVE or QUARANTINED via OpenMetadata UI or REST API
4
Runtime Enforcement
Backend polls/receives webhook → calls set_masked_heads() → HeadMaskStore updates → subsequent generations apply mask
🔧 Engineering Robustness
Tagging includes retry/backoff logic. Prefers column-level tags, falls back to table-level when needed. Webhook handler validates secrets before processing.
